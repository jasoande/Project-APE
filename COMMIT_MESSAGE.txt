Remove service account authentication - OAuth-only

BREAKING CHANGE: Service account authentication has been completely removed.
All Google Drive authentication now uses OAuth 2.0 exclusively.

## Summary

Project APE now uses OAuth 2.0 as the only authentication method for Google
Drive access. Service account support has been completely removed from code,
scripts, and documentation.

## Changes

### Core Code
- Remove service account authentication from drive_manager.py
  - Delete `from google.oauth2 import service_account` import
  - Remove `_service_account_authenticate()` method (25 lines)
  - Remove auth_method config option
  - Remove service_account_key config option
  - Update docstrings to OAuth-only

- Update config_generator.py to OAuth-only defaults
  - Remove 'auth_method': 'service_account' from DRIVE_CONFIG
  - Remove 'service_account_key': None from config templates

### Files Deleted
- service-account-key.json (empty credentials file)
- developer-docs/create-service-account.sh (service account creator)
- developer-docs/test-service-account-quick.sh (service account tester)
- developer-docs/save/service-account-key.json (backup)
- developer-docs/save/service-account.json (backup)

### Documentation
- README.md
  - Remove service account setup instructions
  - Remove create-service-account.sh references
  - Update file structure to show OAuth-only credentials

- API_REFERENCE.md
  - Remove GOOGLE_APPLICATION_CREDENTIALS environment variable
  - Remove service-account-key.json from file listings
  - Change "OAuth and service account" to "OAuth 2.0 user authentication"

### Scripts (Partial)
- setup.sh
  - Remove service account setup section (lines 234-282)
  - Change from if/else auth method to OAuth-only flow
