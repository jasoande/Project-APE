## Fix #2: Service Account Permission Error

**Problem:** Container couldn't read service-account-key.json
```
PermissionError: [Errno 13] Permission denied: '/app/service-account.json'
```

**Root cause:** UID mismatch
- Host: UID 501 (macOS user)
- Container: UID 1000 (apeuser)
- File: 600 permissions

**Solution:** Added `--userns=keep-id` to Podman
- Maps host UID → container UID
- Container becomes UID 501, can read files
- Maintains 600 security

**Files changed:**
- launch_ape.sh (added --userns=keep-id)
- setup-credentials.sh (added --userns=keep-id)
- setup.sh (auto-removes old credentials volume)

**Impact:** Fixes authentication failures

## Fix #3: Always Upload PDF

**Problem:** PDF upload skipped if notebook exists

**Old logic:**
```python
if consolidated_pdf and not is_existing:
    self._upload_pdf(consolidated_pdf)
elif is_existing:
    logger.info("Notebook exists, skipping PDF upload")
```

**New logic:**
```python
if consolidated_pdf:
    logger.info(f"Uploading PDF (existing: {is_existing})")
    self._upload_pdf(consolidated_pdf)
```

**Files changed:**
- core/client_pipeline.py (removed is_existing check)

**Impact:** 
- More reliable (PDF always present)
- Slightly slower (+5-10s per run)

## Fix #6: ARM Architecture Support

**Problem:** Google Cloud SDK repo only has x86_64 packages
```
Error: package google-cloud-cli-573.0.0-1.x86_64 does not have a compatible architecture
```

**Solution:** Detect architecture and use appropriate install method

**Logic:**
```bash
ARCH=$(uname -m)
if [[ "$ARCH" == "x86_64" ]]; then
    # Use RPM repo
    sudo dnf install google-cloud-cli
else
    # Download ARM tarball
    curl -O google-cloud-cli-linux-arm.tar.gz
    tar -xf google-cloud-cli-linux-arm.tar.gz
    sudo mv google-cloud-sdk /opt/
fi
```

**Files changed:**
- setup-environment.sh (added ARM detection)

**Impact:** Works on ARM systems (Apple Silicon VM, AWS Graviton, etc.)
