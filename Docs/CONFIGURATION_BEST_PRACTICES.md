## Configuration Overview

### Configuration Methods

**1. Web UI Configuration (Recommended)**
- Browser-based form at http://localhost:8765/configure
- Real-time validation
- Auto-generates `vars.py`
- Preserves existing clients when adding new ones

**2. Manual vars.py Editing**
- Direct Python file editing
- More control over advanced settings
- Required for timing optimization
- Version control friendly

### Configuration File Structure

```python
# vars.py - Generated by Web UI or manually created

# Client list
clients = ["acme_corp", "techco", "pharma_client"]

# Client 1: Acme Corporation
acme_corp_name = "Acme Corporation"
acme_corp_folder = "https://drive.google.com/drive/folders/1A2B3C4D5E..."
acme_corp_industry = "technology"
acme_corp_subsegments = "cloud infrastructure, SaaS platforms, cybersecurity"

# Client 2: TechCo
techco_name = "TechCo Industries"
techco_folder = "https://drive.google.com/drive/folders/1F7G8H9I0J..."
techco_industry = ""  # Auto-detect
techco_subsegments = "AI/ML, data analytics, IoT"

# Global Settings
persona = "Enterprise technology solutions architect"
MODE = "fast"  # or "deep"
DASHBOARD_PORT = 8765

# Timing Configuration (Fast Mode)
TIMINGS = {
    'ask_prompt_delay': (8.0, 12.0),
    'chat_prompt_delay': (5.0, 8.0),
    'source_processing_wait': (30, 45),
    'initial_offset': (0, 30),
}

# Timing Configuration (Deep Mode)
DEEP_TIMINGS = {
    'ask_prompt_delay': (15.0, 25.0),
    'chat_prompt_delay': (10.0, 15.0),
    'source_processing_wait': (45, 60),
    'initial_offset': (0, 30),
}

# Retry Configuration
RETRY_CONFIG = {
    'max_attempts': 5,
    'base_delay': 10,
    'ask_max_attempts': 7,
    'ask_base_delay': 30,
}
```

## Client Configuration Best Practices

### Naming Conventions

**Client IDs:**
```python
# ✅ Good: lowercase, underscores, descriptive
clients = ["acme_corp", "techco_industries", "pharma_client"]

# ❌ Bad: spaces, special characters, too long
clients = ["Acme Corp", "tech-co!", "pharmaceutical_company_very_long_name_2024"]
```

**Client Names:**
```python
# ✅ Good: proper case, official name
acme_corp_name = "Acme Corporation"
techco_name = "TechCo Industries Inc."

# ❌ Bad: all caps, abbreviations, informal
acme_corp_name = "ACME CORP"
techco_name = "TC Inc"
```

### Drive Folder URLs

**Best Practices:**
```python
# ✅ Good: Full Drive folder URL
client_folder = "https://drive.google.com/drive/folders/1A2B3C4D5E6F7G8H9I0J"

# ✅ Acceptable: Folder ID only (auto-constructed)
client_folder = "1A2B3C4D5E6F7G8H9I0J"

# ❌ Bad: Local path (legacy, not portable)
client_folder = "/Users/jason/Documents/ClientName"

# ❌ Bad: Invalid URL format
client_folder = "https://drive.google.com/file/d/..." # File URL, not folder
```

**Folder Organization:**
```
Google Drive Structure:
├── Project-APE-Clients/
│   ├── Acme-Corp/
│   │   ├── Acme-Annual-Report-2025.pdf

│   │   └── ... (5-15 PDFs recommended)
```

## Timing and Performance Tuning

### Understanding Timing Parameters

**Delay Ranges:**
```python
# Format: (min_seconds, max_seconds)
# Actual delay = random value between min and max

'ask_prompt_delay': (8.0, 12.0)
# Waits 8-12 seconds between research queries
# Prevents API rate limiting
```

### Fast Mode Optimization

**Balanced (Default):**
```python
TIMINGS = {
    'ask_prompt_delay': (8.0, 12.0),      # Research queries
    'chat_prompt_delay': (5.0, 8.0),      # Analysis prompts
    'source_processing_wait': (30, 45),   # After PDF upload
    'initial_offset': (0, 30),            # Anti-thundering-herd
}
# Expected: 15-20 min, 5-10% retry rate
```

**Aggressive (Speed Priority):**
```python
TIMINGS = {
    'ask_prompt_delay': (5.0, 8.0),       # Faster research
    'chat_prompt_delay': (3.0, 5.0),      # Faster analysis
    'source_processing_wait': (25, 35),   # Shorter wait
    'initial_offset': (0, 20),
}
# Expected: 12-15 min, 10-20% retry rate
# ⚠️ Higher retry rate acceptable for speed
```

**Conservative (Quality Priority):**
```python
TIMINGS = {
    'ask_prompt_delay': (12.0, 18.0),     # More time for API
    'chat_prompt_delay': (8.0, 12.0),     # Avoid rate limits
    'source_processing_wait': (45, 60),   # Full processing time
    'initial_offset': (0, 40),
}
# Expected: 20-25 min, <3% retry rate
# ✅ Use when API quota is constrained
```

### Deep Mode Optimization

**Balanced (Default):**
```python
DEEP_TIMINGS = {
    'ask_prompt_delay': (15.0, 25.0),
    'chat_prompt_delay': (10.0, 15.0),
    'source_processing_wait': (45, 60),
    'initial_offset': (0, 30),
}
# Expected: 45-60 min, 20-30% retry rate
```

**Maximum Thoroughness:**
```python
DEEP_TIMINGS = {
    'ask_prompt_delay': (25.0, 35.0),     # Allow extensive research
    'chat_prompt_delay': (15.0, 20.0),    # Deep analysis
    'source_processing_wait': (60, 90),   # Full source processing
    'initial_offset': (0, 60),
}
# Expected: 60-80 min, 10-15% retry rate
# ✅ Best quality, acceptable for critical deals
```

### Retry Configuration

**Standard (Default):**
```python
RETRY_CONFIG = {
    'max_attempts': 5,           # General operations
    'base_delay': 10,            # Initial retry wait
    'ask_max_attempts': 7,       # Research queries (quota-sensitive)
    'ask_base_delay': 30,        # Longer wait for quota reset
}
```

**Aggressive (Faster Failure):**
```python
RETRY_CONFIG = {
    'max_attempts': 3,           # Fail faster
    'base_delay': 5,
    'ask_max_attempts': 5,
    'ask_base_delay': 20,
}
# ⚠️ Use only for quick tests, not production
```

**Conservative (High Reliability):**
```python
RETRY_CONFIG = {
    'max_attempts': 7,           # More retries
    'base_delay': 15,
    'ask_max_attempts': 10,      # Maximum persistence
    'ask_base_delay': 45,        # Longer quota wait
}
# ✅ Use for critical workflows where success is mandatory
```

## Resource Planning

### Memory Requirements

**Per-Client Memory Usage:**
```python
# Fast Mode
base_memory = 500  # MB (Python runtime, Flask)
per_client = 300   # MB (PDF processing, API calls)

# Example: 3 clients
total = base_memory + (3 * per_client)
# = 500 + 900 = 1.4 GB

# Deep Mode (more sources)
per_client_deep = 450  # MB
```

**Memory Planning:**
```python
def calculate_memory_needed(num_clients, mode="fast"):
    base = 500  # MB
    per_client = 300 if mode == "fast" else 450
    total_mb = base + (num_clients * per_client)
    return f"{total_mb} MB ({total_mb/1024:.1f} GB)"

# Examples
calculate_memory_needed(1, "fast")   # 800 MB (0.8 GB)
calculate_memory_needed(3, "fast")   # 1400 MB (1.4 GB)
calculate_memory_needed(5, "fast")   # 2000 MB (2.0 GB)
calculate_memory_needed(3, "deep")   # 1850 MB (1.9 GB)
```

### Disk Space Requirements

**Per-Client Disk Usage:**
```
Input PDFs:            50-200 MB  (5-15 files)
Consolidated PDF:      60-250 MB  (merged with TOC)
Generated Analysis:    1-5 MB     (text output)
Logs:                  5-20 MB    (detailed execution logs)
Status Files:          50-200 KB  (JSON status)
Drive Cache (7 days):  50-200 MB  (cached downloads)

Total per client:      200-700 MB
```

**Disk Planning:**
```python
# 100 clients over 30 days
clients = 100
avg_per_client = 400  # MB
retention_days = 30

disk_needed_gb = (clients * avg_per_client * retention_days) / 1024 / 30
# ≈ 40 GB for 100 clients/month with 30-day retention
```

### Network Bandwidth

**Per-Workflow Network Usage:**
```
Fast Mode:
  - PDF downloads:     50-200 MB
  - API requests:      20-50 MB (NotebookLM, Drive)
  - Source imports:    100-300 MB (40-80 external sources)
  Total:               170-550 MB per workflow

Deep Mode:
  - PDF downloads:     50-200 MB (same)
  - API requests:      50-100 MB (more calls)
  - Source imports:    300-900 MB (90-180 sources)
  Total:               400-1200 MB per workflow
```

**Bandwidth Planning:**
```python
# 20 clients/day in Fast mode
clients_per_day = 20
avg_bandwidth_mb = 350

daily_bandwidth_gb = (clients_per_day * avg_bandwidth_mb) / 1024
# = 6.8 GB/day

monthly_bandwidth_gb = daily_bandwidth_gb * 30
# = 205 GB/month

# Minimum sustained speed: 10 Mbps recommended
```

## Persona Configuration

### Effective Personas

**Structure: Role + Industry/Focus**
```python
# ✅ Excellent: Specific role + specific domain
persona = "Red Hat solutions architect specializing in financial services cloud migrations"
persona = "Enterprise sales engineer focused on pharmaceutical manufacturing IT modernization"
persona = "Strategic consultant for healthcare digital transformation"

# ✅ Good: Role + general domain
persona = "Technology solutions architect"
persona = "Enterprise account executive"

# ⚠️ Acceptable: Generic role
persona = "Solutions architect"

# ❌ Too Vague: No targeting
persona = "Business professional"
```

**Industry-Specific Examples:**

**Technology Sales:**
```python
persona = "Cloud solutions architect specializing in hybrid cloud and containerization"
persona = "DevOps consultant for enterprise CI/CD transformations"
persona = "Cybersecurity sales engineer focused on zero-trust architectures"
```

**Consulting:**
```python
persona = "Management consultant specializing in digital transformation strategy"
persona = "Technology advisor for M&A due diligence in SaaS sector"

```

**Financial Services:**
```python
persona = "Fintech solutions consultant for banking digital transformation"

```

### Persona Impact on Analysis

**Persona influences:**
- Terminology used in analysis (technical vs business language)
- Pain points emphasized (operational vs strategic)
- Technology recommendations (specific vs general)
- ROI framing (cost savings vs revenue growth)

**Example comparison:**

**Persona: "Solutions architect"**
- Focus: Technical architecture, integration patterns, scalability
- Pain points: System complexity, technical debt, performance issues
- Recommendations: Microservices adoption, API strategy, cloud migration

**Persona: "CFO advisor"**
- Focus: Cost optimization, ROI, financial risk
- Pain points: IT spend efficiency, vendor lock-in costs, budget overruns
- Recommendations: TCO reduction, FinOps practices, cloud cost management

## Prompt Customization

### Prompt File Locations

```
Repository Root:
├── ask_prompt_01.txt           # Research query 1
├── ask_prompt_02.txt           # Research query 2
├── chat_prompt_consolidated_01.txt  # Industry + Challenges
├── chat_prompt_consolidated_02.txt  # Technology + Competitive
├── chat_prompt_consolidated_03.txt  # Pain Points + Opportunities
├── chat_prompt_consolidated_04.txt  # Decision Makers + Buying Process
├── chat_prompt_consolidated_05.txt  # Value Prop + Metrics
└── chat_prompt_consolidated_06.txt  # Risks + Recommendations
```

### Variable Substitution

**Available variables:**
```python
$name          # Client name (e.g., "Acme Corporation")
$industry      # Industry (e.g., "pharmaceuticals")
$subsegments   # Subsegments (e.g., "drug discovery, clinical trials")
$persona       # Persona (only in chat prompts, not ask prompts)
```

**Example prompt:**
```text
# ask_prompt_01.txt
Research the $industry industry focusing on $subsegments. 
Analyze trends, disruptions, and key players relevant to $name.
```

**After substitution:**
```text
Research the pharmaceuticals industry focusing on drug discovery, clinical trials.
Analyze trends, disruptions, and key players relevant to Acme Corporation.
```

### Customization Best Practices

**✅ Do:**
- Test prompt changes with single client first
- Keep variable names exactly as shown ($name, not $client_name)
- Maintain professional, objective tone
- Focus on actionable insights
- Reference specific analysis dimensions

**❌ Don't:**
- Make prompts overly long (>500 words)
- Use client-specific hardcoded data (defeats reusability)
- Change variable syntax ($name is required format)
- Remove persona from chat prompts (breaks targeting)

## Multi-Client Strategies

### Parallel Execution Limits

**Recommended configurations:**
```python
# Light load (development/testing)
clients = ["client1"]
MODE = "fast"
# Memory: 800 MB, Time: 15-20 min

# Moderate load (typical production)
clients = ["client1", "client2", "client3"]
MODE = "fast"
# Memory: 1.4 GB, Time: 20 min (parallel)

# Heavy load (batch processing)
clients = ["client1", "client2", "client3", "client4", "client5"]
MODE = "fast"
# Memory: 2.0 GB, Time: 25 min (parallel)
# ⚠️ Maximum recommended

# Deep mode limitation
clients = ["client1", "client2"]
MODE = "deep"
# Memory: 1.4 GB, Time: 60 min (parallel)
# ⚠️ Do not exceed 2-3 clients in deep mode
```

### Batch Processing Strategies

**Strategy 1: Time-based batches**
```python
# Morning batch (high priority)
clients = ["urgent_client1", "urgent_client2", "urgent_client3"]
MODE = "fast"
# Run: 9 AM, complete by 9:25 AM

# Afternoon batch (standard)
clients = ["standard1", "standard2", "standard3", "standard4"]
MODE = "fast"
# Run: 2 PM, complete by 2:30 PM

# Evening batch (deep analysis)
clients = ["deep_client1", "deep_client2"]
MODE = "deep"
# Run: 6 PM, complete by 7:00 PM
```

**Strategy 2: Priority-based execution**
```python
# vars-high-priority.py
clients = ["critical_deal_1", "critical_deal_2"]
MODE = "deep"  # Best quality for critical deals

# vars-standard.py
clients = ["client_a", "client_b", "client_c", "client_d", "client_e"]
MODE = "fast"  # Quick turnaround

# Execute sequentially
./ape-run.sh --vars vars-high-priority.py --mode deep
./ape-run.sh --vars vars-standard.py --mode fast
```
